# テスト作成コマンド

## ワークフロー

### Step 1: テスト対象を確認する

`AskUserQuestion` ツールで以下を確認する：
- テストを追加したいファイル・クラス・機能
- 特に重点的にテストしたい観点（正常系・異常系・エッジケースなど）

### Step 2: 既存のテスト状況を把握する

`test/` ディレクトリを確認し、既存のテストと重複しないよう把握する。

### Step 3: テストを作成する（flutter-tester）

`Task` ツールで `flutter-tester` エージェントを呼び出す：
- subagent_type: flutter-tester
- description: テストコードの作成
- prompt: テスト対象のファイルパスと観点を渡す。以下を含めるよう指示する：
  - ViewModel / UseCase / Repository のユニットテスト（モック使用）
  - 主要 Widget のウィジェットテスト
  - 正常系・異常系・ローディング状態のカバー

### Step 4: テストが通ることを確認する

以下を実行して結果を確認する：

```bash
flutter test
```

失敗したテストがあればフィードバックして修正する。

## 重要な制約

1. **Task ツールを使うこと**: エージェントの呼び出しに bash コマンドを使わない
2. **認証情報をテストに含めない**: PAT・owner/repo はハードコードしない
3. **既存テストを壊さない**: 新規テスト追加後も全テスト pass を確認する

## 完了後の出力

ユーザーに以下を報告する：
- 作成したテストファイル一覧
- テストケース数と内容の概要
- テスト実行結果（pass / fail）
